  <action>
Create `.github/workflows/lint.yml` with the following structure:

**Trigger:** `push` and `pull_request` on `main` branch.

**Jobs:**

### Job 1: `rust-lint`
- Runner: `ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `dtolnay/rust-toolchain@stable` (no extra components needed for clippy/rustfmt — they ship with stable)
  3. Cache Rust: `Swatinem/rust-cache@v2` with `workspaces: src-tauri`
  4. `cargo fmt --all -- --check` (format check)
  5. `cargo clippy --all-targets --all-features -- -D warnings` (lint check, treat warnings as errors)

### Job 2: `ts-lint`
- Runner: `ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `pnpm/action-setup@v4` (reads packageManager from package.json if available, otherwise uses default)
  3. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'pnpm'`
  4. `pnpm install --frozen-lockfile`
  5. `pnpm tsc --noEmit` (TypeScript type-check — build command is `tsc && vite build`, but for lint we only need the type-check pass)
  6. `pnpm eslint . --ext .ts,.tsx` (ESLint check — add `--max-warnings 0` to treat warnings as errors if eslint config exists)

**Concurrency:** Use `concurrency: { group: lint-${{ github.ref }}, cancel-in-progress: true }` to cancel stale runs.
  </action>
  <verify>
    <automated>cat .github/workflows/lint.yml | grep -E "(cargo fmt|cargo clippy|pnpm tsc|eslint)" | wc -l</automated>
  </verify>
  <done>
    - lint.yml exists at .github/workflows/lint.yml
    - Contains rust-lint job with cargo fmt and cargo clippy
    - Contains ts-lint job with TypeScript type-check and ESLint
    - Triggers on push and pull_request to main
    - Uses concurrency group to cancel stale runs
  </done>

  <files>.github/workflows/build-test.yml</files>

  <action>
Create `.github/workflows/build-test.yml` with the following structure:

**Trigger:** `push` and `pull_request` on `main` branch.

**Jobs:**

### Job 1: `test`
- Runner: `ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `dtolnay/rust-toolchain@stable`
  3. `Swatinem/rust-cache@v2` with `workspaces: src-tauri`
  4. `pnpm/action-setup@v4`
  5. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'pnpm'`
  6. `pnpm install --frozen-lockfile`
  7. `cargo test` (Rust unit tests, working-directory: src-tauri)
  8. `pnpm vitest run` (Frontend tests — if vitest config exists, otherwise skip with warning)

### Job 2: `build`
- Strategy: matrix with `platform: [ubuntu-22.04, windows-latest]`
- Steps (platform-aware):
  1. `actions/checkout@v4`
  2. Ubuntu only: `sudo apt-get update && sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf` (Tauri v2 Linux dependencies)
  3. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'pnpm'`
  4. `pnpm/action-setup@v4`
  5. `dtolnay/rust-toolchain@stable`
  6. `Swatinem/rust-cache@v2` with `workspaces: src-tauri`
  7. `pnpm install --frozen-lockfile`
  8. Build with `tauri-apps/tauri-action@v0` (use `tagName` and `releaseName` only if creating a release — for CI-only builds, set `tagName: __NO_RELEASE__` and omit release body). Pass `args: --target ${{ matrix.platform == 'ubuntu-22.04' && 'x86_64-unknown-linux-gnu' || 'x86_64-pc-windows-msvc' }}` but for CI verification, just run `pnpm tauri build` — the tauri-action is mainly for release builds. Actually, for CI-only build verification, use direct commands:
     - `pnpm build` (frontend build)
     - `cargo build --release` in src-tauri OR `pnpm tauri build` (full Tauri build)
  9. Upload artifacts: `actions/upload-artifact@v4` for the built installers (use path patterns: `src-tauri/target/release/bundle/**/*`)

**Concurrency:** Use `concurrency: { group: build-${{ github.ref }}, cancel-in-progress: true }`
  </action>
  <verify>
    <automated>cat .github/workflows/build-test.yml | grep -E "(cargo test|vitest|tauri build|platform)" | wc -l</automated>
  </verify>

    - Contains test job running cargo test and vitest
    - Contains build job with matrix for ubuntu and windows
    - Installs required Linux Tauri dependencies
    - Uploads build artifacts
    - Triggers on push and pull_request to main
  </done>
