## Release Targets

| OS      | Arch   | Installers       | Runner               |
|---------|--------|------------------|----------------------|
| Linux   | x86_64 | AppImage, .deb   | ubuntu-22.04         |
| Linux   | arm64  | AppImage, .deb   | ubuntu-22.04-arm     |
| Windows | x86_64 | .msi, .nsis.exe  | windows-latest       |
| Windows | arm64  | .msi, .nsis.exe  | windows-latest (cross-compile with --target aarch64-pc-windows-msvc) |
| macOS   | x86_64 | .dmg             | macos-13 (Intel)     |
| macOS   | arm64  | .dmg             | macos-latest (M1/M2) |

## GitHub Actions Considerations

- `ubuntu-22.04-arm` runners are available on GitHub (ARM64 native, free for public repos)
- Windows ARM64 cross-compile: use `--target aarch64-pc-windows-msvc` with rustup target add
- macOS: separate runners for Intel (macos-13) and Apple Silicon (macos-latest)
- Tauri v2 `tauri-apps/tauri-action@v0` handles build + bundle per platform
- No signing: skip TAURI_PRIVATE_KEY and TAURI_SIGNING_PRIVATE_KEY setup

  <action>
Create `.github/workflows/release.yml` with the following structure:

**Trigger:** `push` tags matching `v*` pattern ONLY (not on every push). Use `on: push: tags: ['v*']`.

**Permissions:** `contents: write` (for creating releases).

### Job 1: `build` (matrix strategy)
```yaml
strategy:
  fail-fast: false
  matrix:
    include:
      - platform: ubuntu-22.04
        rust_target: x86_64-unknown-linux-gnu
        label: Linux x86_64
      - platform: ubuntu-22.04-arm
        rust_target: aarch64-unknown-linux-gnu
        label: Linux ARM64
      - platform: windows-latest
        rust_target: x86_64-pc-windows-msvc
        label: Windows x86_64
      - platform: macos-13
        rust_target: x86_64-apple-darwin
        label: macOS Intel
      - platform: macos-latest
        rust_target: aarch64-apple-darwin
        label: macOS Apple Silicon
```

Steps per matrix entry:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'pnpm'`
3. `pnpm/action-setup@v4`
4. `dtolnay/rust-toolchain@stable` with `targets: ${{ matrix.rust_target }}`
5. `Swatinem/rust-cache@v2` with `workspaces: src-tauri`
6. Ubuntu only: install system deps (`libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf`)
7. Windows only: `rustup target add aarch64-pc-windows-msvc` (for ARM64 cross-compile)
8. `pnpm install --frozen-lockfile`
9. Build: `pnpm tauri build --target ${{ matrix.rust_target }}`
   - NOTE: Since no code signing is needed, do NOT set TAURI_PRIVATE_KEY or TAURI_SIGNING_PRIVATE_KEY. Tauri will produce unsigned installers.
10. Generate checksums for each artifact:
    ```bash
    cd src-tauri/target/${{ matrix.rust_target }}/release/bundle
    for f in **/*.{AppImage,deb,msi,exe,dmg}; do
      sha256sum "$f" > "${f}.sha256"
    done
    ```
    For macOS the path structure may be different — handle `bundle/macos/*.dmg` and `bundle/dmg/*.dmg`.
11. `actions/upload-artifact@v4` — upload all files from `src-tauri/target/${{ matrix.rust_target }}/release/bundle/` with a matrix-specific artifact name like `clawstation-${{ matrix.label }}`

### Job 2: `release` (depends on `build`)
- Runner: `ubuntu-latest`
- Needs: `build`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/download-artifact@v4` — download all artifacts to `artifacts/` directory
  3. Generate checksums.txt — combine all .sha256 files into a single checksums.txt at the root:
     ```bash
     find artifacts -name "*.sha256" -exec cat {} \; > checksums.txt
     ```
  4. Generate changelog from git commits since last tag:
     ```bash
     PREV_TAG=$(git describe --tags --abbrev=0 HEAD^ 2>/dev/null || echo "")
     if [ -n "$PREV_TAG" ]; then
       git log ${PREV_TAG}..HEAD --oneline --no-merges > changelog.txt
     else
       git log --oneline --no-merges > changelog.txt
     fi
     ```
  5. `softprops/action-gh-release@v2` — create release with:
     - `files: artifacts/**/*` (all installer files + checksums)
     - `body_path: changelog.txt`
     - `generate_release_notes: true` (GitHub auto-generates additional notes)
     - `draft: false`
     - `prerelease: ${{ contains(github.ref, '-beta') || contains(github.ref, '-rc') }}`
     - `tag_name: ${{ github.ref_name }}`

**Concurrency:** `concurrency: { group: release-${{ github.ref }}, cancel-in-progress: false }` — NEVER cancel releases in progress.
  </action>
  <verify>
    <automated>cat .github/workflows/release.yml | grep -E "(v\*|sha256|action-gh-release|ubuntu-22.04-arm|macos-13|macos-latest|windows-latest)" | wc -l</automated>
  </verify>
  <done>
    - release.yml exists at .github/workflows/release.yml
    - Triggers ONLY on v* tags
    - Matrix covers 5 platform/arch combinations: Linux x64, Linux ARM64, Windows x64, macOS Intel, macOS Apple Silicon
    - SHA256 checksums generated per artifact
    - All artifacts uploaded to GitHub Release via softprops/action-gh-release
    - Changelog auto-generated from git commits
    - No code signing configured (unsigned installers)
    - Concurrency group does NOT cancel in-progress releases
  </done>

Create `.github/workflows/ci.yml` that combines lint.yml and build-test.yml into a single unified CI workflow. Then delete the separate lint.yml and build-test.yml files.

**Trigger:** `push` to `main` and `pull_request` to `main`.

**Structure:**

### Job 1: `lint` (fast gate)
- Runner: `ubuntu-latest`
- Runs in parallel: rust-lint and ts-lint as separate steps (not separate jobs — saves runner overhead)
- Steps:
  1. `actions/checkout@v4`
  2. `dtolnay/rust-toolchain@stable`
  3. `Swatinem/rust-cache@v2` with `workspaces: src-tauri`
  4. `pnpm/action-setup@v4`
  5. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'pnpm'`
  6. `pnpm install --frozen-lockfile`
  7. `cargo fmt --all -- --check` in src-tauri
  8. `cargo clippy --all-targets --all-features -- -D warnings` in src-tauri
  9. `pnpm tsc --noEmit`
  10. `pnpm eslint . --ext .ts,.tsx --max-warnings 0` (only if eslint config exists — otherwise skip with echo)

### Job 2: `test` (depends on lint)
- Runner: `ubuntu-latest`
- Needs: `lint`
- Steps:
  1. `actions/checkout@v4`
  2. `dtolnay/rust-toolchain@stable`
  3. `Swatinem/rust-cache@v2` with `workspaces: src-tauri`
  4. `pnpm/action-setup@v4`
  5. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'pnpm'`
  6. `pnpm install --frozen-lockfile`
  7. `cargo test` in src-tauri
  8. `pnpm vitest run` (continue-on-error: true if no tests exist yet — once tests are added, remove continue-on-error)

### Job 3: `build` (depends on test)
- Strategy: matrix `platform: [ubuntu-22.04, windows-latest]`
- Needs: `test`
- Same structure as build-test.yml build job but only for CI verification (not release)
- Steps per matrix entry:
  1. `actions/checkout@v4`
  2. Platform-specific system deps (Linux only)
  3. Node + pnpm + Rust setup with caching
  4. `pnpm install --frozen-lockfile`
  5. `pnpm tauri build` (full build, no release)
  6. `actions/upload-artifact@v4` for `src-tauri/target/release/bundle/**/*`

**Concurrency:** `concurrency: { group: ci-${{ github.ref }}, cancel-in-progress: true }`

**Delete:** Remove `.github/workflows/lint.yml` and `.github/workflows/build-test.yml` since ci.yml replaces them.
  </action>

    - Build job runs on ubuntu and windows matrix

```yaml
version: 2
updates:
  - package-ecosystem: "cargo"
    directory: "/src-tauri"

      - "dependencies"
      - "rust"

      prefix: "deps(rust):"
    groups:
      tauri:
        patterns:
          - "tauri*"
      rust-minor:
        update-types:
          - "minor"
          - "patch"

  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
    labels:
      - "dependencies"
      - "javascript"
    commit-message:
      prefix: "deps(node):"
    groups:
      tauri:
        patterns:
          - "@tauri-apps/*"
      react:
        patterns:
          - "react"
          - "react-dom"
          - "@types/react*"
      minor:
        update-types:
          - "minor"
          - "patch"
```
