# Plan 01: Foundation — Project Scaffold

## Objective

Scaffold the Tauri v2 project with React 19 frontend, install all Phase 1 dependencies (frontend + Rust), and configure Tauri capabilities for security permissions. This creates the working project foundation that all subsequent plans build on.

**Purpose:** Establish a compiling, launchable Tauri v2 app with correct dependency versions and security configuration.

**Output:** A greenfield Tauri v2 project that `pnpm tauri dev` can launch, with all Phase 1 plugins registered and capabilities configured.

<task type="auto">
  <name>Task 1: Scaffold Tauri v2 Project with React-TS Template</name>
  <files>package.json, src-tauri/Cargo.toml, src-tauri/src/lib.rs, src-tauri/tauri.conf.json, vite.config.ts, tsconfig.json, src/main.tsx, src/App.tsx, src/index.css</files>
  <action>
    Run `npm create tauri-app@latest openclaw-desktop -- --template react-ts` to scaffold the project. After scaffolding:
    1. Switch to pnpm: run `pnpm install` in the project directory
    2. Verify `@tauri-apps/api` version matches `src-tauri/Cargo.toml` tauri version (both should be 2.10.x)
    3. Check Vite version in `package.json`. If Vite 6, upgrade to Vite 8 and ensure `@vitejs/plugin-react` is v6+
    4. Set app identifier to `com.openclaw.installer` in `src-tauri/tauri.conf.json`
    Important gotchas from RESEARCH.md:
    - Use `com.openclaw.installer` (not `.app` suffix — conflicts with macOS conventions)
    - If Vite 8 + `@vitejs/plugin-react` v6 — ensure `"onlyBuiltDependencies": ["esbuild"]` in package.json
  </action>
  <verify>
    <automated>pnpm install && pnpm tauri build 2>&1 | tail -5</automated>
  </verify>
  <done>Project compiles with `pnpm tauri build` and launches with `pnpm tauri dev`. App identifier is `com.openclaw.installer`.</done>
</task>

<task type="auto">
  <name>Task 2: Install Frontend Dependencies</name>
  <files>package.json, pnpm-lock.yaml</files>
  <action>
    Install all Phase 1 frontend dependencies:
    ```
    pnpm add @tauri-apps/api @tauri-apps/plugin-os @tauri-apps/plugin-shell @tauri-apps/plugin-store @tauri-apps/plugin-notification zustand @tanstack/react-query lucide-react react-router-dom sonner
    pnpm add -D @tauri-apps/cli tailwindcss @tailwindcss/vite typescript vitest @testing-library/react @testing-library/jest-dom
    ```
    Why these specific packages:
    - `@tauri-apps/plugin-os` — Platform detection (PLAT-01, PLAT-02). Frontend-only API, no custom Rust command needed.
    - `@tauri-apps/plugin-shell` — Register now for Phase 2 Docker operations. Not actively used in Phase 1.
    - `@tauri-apps/plugin-store` — Persistent key-value store for app settings. Register now.
    - `@tauri-apps/plugin-notification` — System notifications for Phase 3+. Register now.
    - `zustand` — UI state (sidebar, theme). NOT for async data (that's TanStack Query).
    - `@tanstack/react-query` — Async state for all `invoke()` results.
    - `lucide-react` — Icon library per UI-SPEC.md.
    - `react-router-dom` — Client-side routing. Use `HashRouter` for Tauri compatibility.
    - `sonner` — Toast notifications for ERR-01 error display.
    - `vitest` + `@testing-library/react` — Test framework for validation.
  </action>
  <verify>
    <automated>pnpm ls zustand @tanstack/react-query react-router-dom lucide-react sonner @tauri-apps/plugin-os 2>&1 | grep -E "(zustand|tanstack|react-router|lucide|sonner|plugin-os)"</automated>
  </verify>
  <done>All 13 frontend dependencies installed. `pnpm ls` shows correct versions. No peer dependency warnings.</done>
</task>

<task type="auto">
  <name>Task 3: Configure Rust Dependencies and Tauri Capabilities</name>
  <files>src-tauri/Cargo.toml, src-tauri/src/lib.rs, src-tauri/capabilities/default.json</files>
  <action>
    Step A — Add Rust crates to `src-tauri/Cargo.toml`:
    ```
    cd src-tauri && cargo add tauri-plugin-os tauri-plugin-shell tauri-plugin-store tauri-plugin-notification
    cargo add thiserror serde --features serde/derive
    cargo add serde_json anyhow tokio --features tokio/full
    ```
    Step B — Register plugins in `src-tauri/src/lib.rs`. Update the `run()` function to call `.plugin()` for each: tauri_plugin_os::init(), tauri_plugin_shell::init(), tauri_plugin_store::init(), tauri_plugin_notification::init().
    Step C — Configure capabilities in `src-tauri/capabilities/default.json`:
    Grant permissions: core:default, core:path:default, core:event:default, core:window:default, core:app:default, core:resources:default, core:menu:default, os:default, shell:default, store:default, notification:default.
    Critical: The `os:default` permission includes allow-arch, allow-platform, allow-family, allow-version, allow-locale, allow-os-type, allow-exe-extension. No need to list individual permissions.
  </action>
  <verify>
    <automated>cd src-tauri && cargo check 2>&1 | tail -5</automated>
  </verify>
  <done>Cargo.toml has all required crates. lib.rs registers all 4 plugins. capabilities/default.json grants correct permissions. `cargo check` passes.</done>
</task>
