### Sidebar Navigation

- Width: 220px
- Icon + label pattern (Lucide icons)
- Active item: blue-600 left border (3px) + subtle blue-50 background
- Inactive items: slate-500 icon/text
- Collapse behavior: not in Phase 1

- Height: 48px
- Left: App logo + "OpenClaw Desktop" (Display text, 20px weight 600)
- Right: Platform indicator badge (Lucide Monitor/Cpu icon + "Windows" or "Linux")

- Border-bottom: 1px solid slate-200

- Padding: 24px (lg)
- Max content width: unconstrained (fills available space)
- Pages render here via router

- Uses shadcn `toaster` component (sonner-based)
- Position: bottom-right
- Error toast: red-600 icon, white background, red-50 left border
- Duration: 5s auto-dismiss, or manual close
- "Details" expandable for technical error (collapsed by default)

## Interaction Contracts

### Platform Detection (PLAT-01, PLAT-02)

- On app launch, Rust backend detects OS via `std::env::consts::OS`
- Result sent to frontend via Tauri command `get_platform_info`
- PlatformBadge updates reactively via TanStack Query
- Supported: `windows`, `linux` → badge shows "Windows" or "Linux"
- Unsupported fallback: badge shows "Unknown" + muted styling

### Error Handling (ERR-01)

- All Tauri commands return `Result<T, AppError>`
- `AppError` struct contains: `code: string`, `message: string`, `suggestion: string`
- Frontend maps error codes to plain-language messages via error map
- Error displayed as toast (sonner) with:
  - Human-readable message (from error map or AppError.message)
  - Suggested fix (AppError.suggestion) — shown on expand
  - "Dismiss" button

### Sidebar Navigation

- Active route highlighted with accent color indicator
- Click navigates via React Router
- No page transitions in Phase 1 (instant swap)
- All nav items are always visible (no permission gating in Phase 1)

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Sidebar keyboard nav | Arrow keys navigate items, Enter activates, focus-visible ring |
| Skip-to-content link | Hidden link in header, visible on focus |
| Error announcements | Toasts use `role="alert"` + `aria-live="assertive"` |
| Color contrast | All text meets WCAG AA (4.5:1 body, 3:1 large text) — verified with slate tokens |
| Focus management | Route changes focus the content area heading |
| Reduced motion | Respect `prefers-reduced-motion` — disable transitions |
