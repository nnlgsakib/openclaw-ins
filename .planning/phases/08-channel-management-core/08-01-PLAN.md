# Phase 8: Channel Management Core - Plan

**Created:** 2026-03-26
**Status:** In Progress
**Requirements:** CHAN-01, CHAN-02, CMON-01

## Goal

Build channel overview UI and connection infrastructure.

## Success Criteria

1. User sees list of all available channels (WhatsApp, Telegram, Discord, Slack) with status badges
2. User can connect/disconnect channels from the UI
3. App detects expired sessions and prompts for reconnection
4. Channel page is accessible from sidebar navigation

## Task Breakdown

### P01: Backend — Channel Tauri Commands

**Files to create:**
- `src-tauri/src/commands/channels.rs`

**Files to modify:**
- `src-tauri/src/commands/mod.rs` — add `pub mod channels;`
- `src-tauri/src/lib.rs` — add commands to `generate_handler!`

**Implementation:**
- `ChannelInfo` struct: `id`, `name`, `channel_type` (whatsapp/telegram/discord/slack), `status` (connected/disconnected/expired/connecting), `last_active_at` (Option<String>)
- `get_channels` command: queries OpenClaw API for channel list, returns `Vec<ChannelInfo>`
- `disconnect_channel` command: calls OpenClaw API to disconnect a channel by ID
- `connect_channel` command: calls OpenClaw API to initiate channel connection
- Uses reqwest to call OpenClaw API (same pattern as monitoring commands)
- Graceful degradation: returns empty vec on failure (consistent with monitoring pattern)

### P02: Frontend — Channel Types & Hook

**Files to create:**
- `src/hooks/use-channels.ts`

**Implementation:**
- TypeScript types mirroring Rust structs: `ChannelInfo`, `ChannelType`, `ChannelStatus`
- `useChannels()` hook: TanStack Query with `["channels"]` key, 30s refetchInterval
- `useDisconnectChannel()` mutation hook
- `useConnectChannel()` mutation hook
- Adaptive polling: 30s when any channel expired, 60s when all healthy

### P03: Frontend — Channel Page

**Files to create:**
- `src/pages/channels.tsx`

**Implementation:**
- Page header: "Channels" title + description + Refresh button
- Card grid (one per channel type: WhatsApp, Telegram, Discord, Slack)
- Each card: channel name + icon, status badge, last activity, action button
- Status badge colors: green (connected), gray (disconnected), yellow (expired), blue (connecting)
- Skeleton loading during fetch (4 skeleton cards)
- Empty/not-running state with informative message
- Disconnect confirmation via AlertDialog
- Toast notification for session expiration

### P04: Integration — Route & Sidebar

**Files to modify:**
- `src/router.tsx` — add `/channels` route
- `src/components/layout/sidebar-nav.tsx` — add Channels nav item with MessageSquare icon
