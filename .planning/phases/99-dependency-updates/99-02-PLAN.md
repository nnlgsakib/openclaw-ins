## Objective

Upgrade 4 major-version Rust dependencies from Dependabot Cargo PRs and fix any API-breaking changes.

**Purpose:** Dependabot opened PRs for sysinfo (0.33→0.38), getrandom (0.3→0.4), reqwest (0.12→0.13), and tokio-tungstenite (0.26→0.29). These are major bumps requiring Cargo.toml changes and potential code fixes.

**Output:** Updated Cargo.toml + Cargo.lock with migrated code that compiles and works.

### Migration Notes

**sysinfo 0.33→0.38:**
- `Disks::new()` → `Disks::new_with_refreshed_list()` (auto-refreshes at creation, no separate `refresh()` needed)
- `System::new()` + `refresh_memory()` → `System::new_all()` or keep `System::new()` + `refresh_memory()` (still works)
- `available_space()` and `available_memory()` signatures unchanged
- Impact: `system_check.rs` — 2 files affected, 3 API calls

**getrandom 0.3→0.4:**
- `getrandom::fill(&mut buf)` — same signature, returns `Result<(), Error>`
- Impact: `docker_install.rs` — 1 line affected

**reqwest 0.12→0.13:**
- Core API unchanged: `Client::new()`, `Client::builder()`, `.get()`, `.post()`, `.json()`, `.send()`, `.json::<T>()`
- TLS backend may change (rustls vs native-tls)
- Impact: 5 files, 15 usages — likely no code changes needed

**tokio-tungstenite 0.26→0.29:**
- Core API unchanged: `connect_async`, `Message`, `MaybeTlsStream`, `WebSocketStream`
- May need feature flag updates
- Impact: 2 files, 4 usages — likely no code changes needed
